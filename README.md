# 极验验证码插件 (Geetest v4)

> 作者：RuiNexus | 版本：1.1.2 | 适用系统：智简魔方财务系统

---

## 简介

本插件通过极验 v4 人机验证替代系统原生图形验证码，采用 **hook 拦截 + 原生表单提交** 的方式，**无需修改任何系统模板或控制器代码**，安装即用。

### 核心原理

```
用户完成极验验证 → 前端自动将极验参数注入表单隐藏字段 → 原生表单提交
→ 系统 captcha_check() → custom_captcha_check 钩子拦截
→ 调用极验 API 二次校验 → 返回结果
```

## 安装

### 1. 前置要求

- 智简魔方财务系统
- 系统已开启验证码功能：后台 → 系统设置 → 基础设置 → 开启「图形验证码」
- [极验官网](https://www.geetest.com/) 注册账号，进入「行为验证 4.0」→ 业务管理中创建应用

### 2. 获取极验凭证

登录 [极验管理后台](https://console.geetest.com/)：

1. 进入 **行为验证 4.0** → 业务管理 → 创建应用
2. 在应用配置中获取：
   - `captcha_id` — 应用 ID
   - `captcha_key` — 密钥（请妥善保管，勿泄露）

### 3. 安装插件

将 `geetest` 文件夹上传至服务器：

```
public/plugins/addons/geetest/
```

然后在后台 → 插件管理 中找到「极验验证码」，点击安装。

### 4. 配置插件

进入 **插件管理 → 极验验证码 → 验证配置** 页面：

| 配置项 | 说明 |
|--------|------|
| captcha_id | 填入从极验后台获取的 captcha_id |
| captcha_key | 填入从极验后台获取的 captcha_key（不修改时留空即可） |
| 展现形式 | float=浮动式 / popup=弹出式 / bind=隐藏式 |
| 验证类型 | 滑动验证等 |
| 替换原生验证码 | 开启=前端显示极验 / 关闭=恢复原生验证码 |
| 禁用原生验证码 | 开启=必须完成极验，禁用原生验证码校验 |

保存后立即生效，**无需清除缓存或重启服务**。

## 配置说明

### 替换原生验证码

- **开启（默认）**：前端所有含验证码的页面自动显示极验按钮，隐藏原生图形验证码
- **关闭**：恢复显示系统原生图形验证码，极验完全不注入。适用于临时测试或排查问题

### 禁用原生验证码

- **关闭（默认）**：未完成极验时，降级使用原生图形验证码
- **开启**：无论前端如何，后端强制要求极验参数。即使用户绕过前端直接提交原生验证码也会被拒绝，返回「图形验证码已被禁用，请完成人机验证」

> 推荐生产环境同时开启「替换原生验证码」和「禁用原生验证码」，达到最高安全性。

本插件基于系统 hook 机制运行，**与具体模板无关**。理论上兼容所有使用智简魔方标准表单结构的模板。

当前已在以下模板上测试通过：

- `default` — 系统默认模板

如果您使用的模板出现以下情况，请在 GitHub Issues 反馈：

- 极验按钮位置异常或不显示
- 验证无法正常通过
- 原生验证码未被正确替换

### 如何反馈模板适配问题

请提供以下信息之一：

1. **在 GitHub Issues 中留言**：附上您使用的模板名称或站点 URL

我们会尽快为您适配并发布更新。

## 技术细节

### 使用了 3 个系统 Hook

| Hook | 方法 | 作用 |
|------|------|------|
| `client_area_head_output` | `clientAreaHeadOutput()` | 在 `<head>` 中注入极验 GT4 SDK |
| `client_area_footer_output` | `clientAreaFooterOutput()` | 在页面底部注入初始化脚本（创建容器、绑定事件、AJAX 拦截） |
| `custom_captcha_check` | `customCaptchaCheck()` | 后端拦截 `captcha_check()` 函数，执行极验二次校验 |

### 前端工作流程

1. 自动在登录/注册/安全等页面创建 `#geetest-box` 容器
2. 定位容器到当前活跃表单中（密码输入框后面）
3. 初始化极验验证码
4. 用户完成验证后缓存结果，并将参数作为隐藏字段注入所有表单
5. 表单提交时检查是否已完成验证，未完成则阻止
6. AJAX 请求（获取验证码等）自动附加极验参数

### 后端校验流程

每次有验证请求时，插件向极验服务器发送二次校验请求：

```
POST https://gcaptcha4.geetest.com/validate
Body: lot_number, captcha_output, pass_token, gen_time, captcha_id, sign_token
```

其中 `sign_token` 由 `HMAC-SHA256(lot_number, captcha_key)` 生成。

## 安全说明

- 所有 PHP 变量注入 JavaScript 时均经过 `json_encode()` 转义，防止 XSS
- 与极验服务器通信启用 SSL 证书验证（`CURLOPT_SSL_VERIFYPEER=true`）
- `captcha_key` 保存时留空不会覆盖旧值，防止误清空密钥
- 支持「禁用原生验证码」模式，防止绕过前端直接提交
